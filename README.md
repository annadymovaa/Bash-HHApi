# Работа с API HeadHunter и обработка данных

В этом проекте я реализовала серию скриптов для работы с API HeadHunter, поэтапно обрабатывая и анализируя данные о вакансиях. Каждый этап оформлен в отдельной директории внутри `src`, а структура репозитория отражает логику выполнения заданий.

## Структура проекта

- **src/api_interaction** — получение данных о вакансиях через HeadHunter API.
- **src/json_to_csv** — преобразование JSON в CSV для анализа.
- **src/sorting** — сортировка данных по дате и идентификатору.
- **src/cleaning** — очистка и выделение уровня позиции из названия.
- **src/statistics** — подсчёт уникальных значений и их частот.
- **src/partitioning** — разбиение и объединение данных по датам.

---

## 1. Получение данных о вакансиях (src/api_interaction)

Я написала shell-скрипт, который принимает название вакансии (например, *"data scientist"*) как аргумент, скачивает информацию о первых 20 вакансиях через HeadHunter API с помощью `curl` и сохраняет результат в файл `hh.json`. Данные в файле отформатированы так, чтобы каждое поле было на отдельной строке. Скрипт исполняемый, используется интерпретатор `/bin/sh`.

### Пример команды для запуска скрипта:
```bash
cd src/api_interaction
./hh.sh "data scientist"
```
*Файл результата: `hh.json`*

---

## 2. Преобразование JSON в CSV (src/json_to_csv)

Для удобства анализа я реализовала скрипт, который с помощью `jq` и отдельного фильтра преобразует JSON-файл в CSV. В итоговый файл `hh.csv` попадают только нужные поля: *id*, *created_at*, *name*, *has_test*, *alternate_url*. Первая строка — заголовки.

### Пример команды для запуска скрипта:
```bash
cd src/json_to_csv
./json_to_csv.sh ../api_interaction/hh.json
```
*Файл результата: `hh.csv`*

---

## 3. Сортировка данных (src/sorting)

Далее я написала скрипт для сортировки CSV-файла по колонке *created_at* (по возрастанию), а при совпадении дат — по *id*. Результат сохраняется в `hh_sorted.csv`, заголовки сохраняются.

### Пример команды для запуска скрипта:
```bash
cd src/sorting
./sorter.sh ../json_to_csv/hh.csv
```
*Файл результата: `hh_sorted.csv`*

---

## 4. Очистка и выделение уровня позиции (src/cleaning)

В этом этапе я реализовала скрипт, который анализирует название вакансии и выделяет из него уровень позиции (*Junior*, *Middle*, *Senior*). Если ни один из этих уровней не найден, в поле записывается "-". Если встречается несколько уровней, они сохраняются через слэш. Итоговый файл — `hh_positions.csv`, с заголовками и отсортированными данными.

### Пример команды для запуска скрипта:
```bash
cd src/cleaning
./cleaner.sh ../sorting/hh_sorted.csv
```
*Файл результата: `hh_positions.csv`*

---

## 5. Подсчёт уникальных позиций (src/statistics)

Я написала скрипт, который подсчитывает количество уникальных значений в колонке *name* (уровни позиций) и выводит результат в CSV-файл `hh_uniq_positions.csv` с сортировкой по убыванию частоты.

### Пример команды для запуска скрипта:
```bash
cd src/statistics
./counter.sh ../cleaning/hh_positions.csv
```
*Файл результата: `hh_uniq_positions.csv`*

---

## 6. Разбиение и объединение данных по датам (src/partitioning)

Для работы с большими объёмами данных я реализовала два скрипта:
- **partitioner.sh** — разбивает исходный CSV по уникальным датам из поля *created_at*, создавая отдельные файлы для каждой даты.
- **concatenator.sh** — объединяет все эти файлы обратно в один CSV, сохраняя исходную структуру и порядок данных.

### Пример команды для запуска скрипта:
```bash
cd src/partitioning
./partitioner.sh ../cleaning/hh_positions.csv
./concatenator.sh
```
*Файлы результата: по датам в текущей директории, объединённый файл — `hh_concatenated.csv`*

Все скрипты исполняемые, используют `/bin/sh`.

### Проект демонстрирует навыки работы с API, обработки и анализа данных, а также автоматизации рутинных задач с помощью shell-скриптов.
